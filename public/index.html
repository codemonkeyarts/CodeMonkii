                  Preferences

                  Theme the palette Monkii wears
                  Applies instantly and sticks — no restart. More theming (custom colors, fonts, light mode) is on the roadmap.
                  Ollama models folder where pulled models are stored on disk
                  Set by the OLLAMA_MODELS environment variable — it overrides this preference. Unset it to manage the folder here.
                  Takes effect the next time Monkii starts Ollama (quit Ollama, then relaunch the app).
                  Projects & chats folder where conversations and project settings are saved
                  Set by the MONKII_DATA_DIR environment variable — it overrides this preference.
                  Changing this restarts the app view. Existing chats stay in the old folder — move the JSON files over if you want them in the new one.
                  Skills folder where SKILL.md folders are scanned from
                  Set by the MONKII_SKILLS_DIR environment variable — it overrides this preference.
                  Point this at an existing library (e.g. ~\.claude\skills) to use your Claude Code skills as-is.
                  File access which folders the browser and attachments can read
                    Set by the MONKII_FS_ROOTS environment variable — it overrides this preference.
                    By default Monkii can only read inside your home folder (this includes AppData and dotfiles like .ssh — narrow it to a specific folder if that matters to you). Add a folder to allow attachments from elsewhere (e.g. a projects drive), or allow the whole disk. Changing this restarts the app view.
                    Remote models — OpenRouter optional; off unless you add a key
                    Set by the MONKII_OPENROUTER_KEY environment variable — it overrides this preference.
                    Set by the MONKII_OR_DATA_COLLECTION environment variable — it overrides this preference.
                    This changes where your words go. Chats that use an OpenRouter model send that chat's messages, project instructions, and attachments to openrouter.ai and the model's provider — they leave this machine. Local models are never affected, and nothing goes remote unless you pick a remote model for a chat. The key is stored OS-encrypted and never shown again. While a key is saved, opening this panel or the ☁ model browser checks your credit balance with OpenRouter (no chat data — just the key, cached for a minute). Saving restarts the app view.
                    Data & backup export a backup, or erase everything
                    Backup zips your projects & chats to a folder you choose — cached embeddings aren't included, they rebuild automatically. Erasing removes every project, chat, and cached embedding; skills and these preferences are untouched. Neither can be undone once done — back up first if you want to keep anything.
                    Update check look for newer Ollama releases
                    Set by the MONKII_UPDATE_CHECK environment variable — it overrides this preference.
                    Off by default. This is the only thing Monkii ever sends off your machine — a version check to GitHub, with no data about you or your chats. Enable it to get update notifications. Changing this restarts the app view.

                    Help & FAQ

                    Getting started

                    Open a Project (or hit Quick chat), pick a model at the top, and type. Enter sends; Shift+Enter makes a newline.

                    Attach knowledge

                    Add files or folders with the 📎 in the chat header (just this chat) or the Knowledge panel (the whole project). Attachments are re-read from disk every message, so your latest edits always show up.

                    Big files & long documents

                    A large attachment — a manuscript, a codebase — is embedded on-device, and only the passages relevant to your question are added to the prompt. You'll see an “indexing %” badge while it prepares. It needs a small embedding model, which Monkii offers to download on first run.

                    Skills

                    Drop a SKILL.md folder into your skills directory, or use ✦ Skills → New / Import. Toggle a skill on for a whole project, or invoke one for a single message by typing / in the composer.

                    Models

                    Pick a model per chat. Manage models… pulls, deletes, and shows disk usage; the ⚙ next to the model opens Model settings (context length, temperature, and more).

                    File access

                    By default Monkii can only read inside your home folder. To attach from elsewhere, widen it in Preferences → File access.

                    Remote models (optional)

                    If your hardware can't run the model you need, add an OpenRouter API key in Preferences → Remote models, then browse the catalog with the ☁ next to the model picker and ★ models into it. Chats using a remote model show a ☁ remote badge — that chat's text goes to OpenRouter and the model's provider. Local chats are never affected.

                    Privacy

                    Out of the box, nothing leaves your machine. Two things are opt-in and off by default: the daily Ollama update check (Preferences → Update check), and remote models — which only apply to chats where you picked a ☁ remote model.

                    Tips

                    • Right-click chats, projects, messages, and skills for quick actions.
                    • Type / in the composer to invoke a skill for one message.
                    • If a chat gets long, Monkii trims old messages to fit — or start a new chat, or raise the context length in Model settings.
                    • If a model keeps restating its plan instead of doing the work ("I'll pull X, then draft Y…" on every reply), "continue" or "progress?" won't break the loop — give it a direct instruction instead, e.g. "Stop planning — write it now."

                    About Monkii

                    🐒 MONKII

                    A local, private LLM studio for Ollama — projects, Claude-style skills, and live file knowledge. Local by default: nothing about you or your chats leaves your machine unless you deliberately point a chat at a ☁ remote model — and Monkii always tells you when the text is leaving the room.

                    Built by Coda Languez as a desktop home for local AI: it began as a ComfyUI-style wrapper around Ollama and grew into a full studio for writing and coding — local-first, and honest about the one thing that isn't: remote models you choose yourself.
